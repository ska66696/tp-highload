## 1. Тема и целевая аудитория
**Discord** - платформа для текстового и голосового общения в рамках пользовательских сообществ (серверов).

### Целевая аудитория
*   **MAU (Monthly Active Users):** 200 млн [^1].
*   **DAU (Daily Active Users):** 26.5 млн [^3].
*   **Количество зарегистрированных аккаунтов:** 560 млн [^2].
*   **География:** Глобальный рынок. Распределение трафика по данным на начало 2026 года [^4]:

| Страна | Доля трафика (%) | Оценка MAU (млн чел) |
| :--- | :--- | :--- |
| США | 29.1% | 58.2 |
| Бразилия | 7.25% | 14.5 |
| Индия | 4.9% | 9.8 |
| Германия | 3.2% | 6.4 |
| Канада | 2.7% | 5.4 |

### Функционал MVP
Основной функционал заключается в создании иерархических сообществ и обеспечении каналов связи внутри них.

1.  **Управление серверами:** создание серверов и текстовых/голосовых каналов.
2.  **Текстовые чаты:** обмен сообщениями в реальном времени внутри каналов.
3.  **Голосовые каналы:** аудио-сессии для группового общения.
4.  **История сообщений:** хранение и доступ к переписке на стороне сервера.
5.  **Статусы:** статусы пользователей («в сети», «играет в...»).
6.  **Вложения (Media):** возможность загрузки изображений и файлов в текстовые каналы.

### Ключевые продуктовые решения
*   Голосовые каналы работают по модели «постоянной комнаты».
*   История сообщений хранится на серверах бессрочно и доступна с любого устройства в любой момент.
*   Синхронизация состояния серверов, сообщений и статусов между всеми активными сессиями пользователя.

## 2. Расчет нагрузки

### Продуктовые метрики
Базовые показатели аудитории, на которые мы опираемся в расчетах:
*   **MAU (Monthly Active Users):** 200 млн [^1]
*   **DAU (Daily Active Users):** 26.5 млн [^3]
*   **Зарегистрированных аккаунтов:** 560 млн [^2]
*   **Активных серверов (Guilds):** 21 млн [^2]

**Средний размер хранилища пользователя**
Discord хранит более 1.5 триллиона сообщений [^5]. Среднее количество на аккаунт: `1.5 трлн / 560 млн ≈ 2678`.
*   *Текст:* Средний размер сообщения составляет ~250 байт [^10].
*   *Вложения:* Опираясь на статистику мессенджеров (WhatsApp [^12]), доля сообщений с медиа составляет ~12%. Для игрового чата примем оценку **10%**. Лимит загрузки для бесплатных пользователей - 10 МБ [^8], средний размер файла примем за 1 МБ.
*   *Профиль:* Аватар (~100 КБ) + настройки.

| Тип данных | Объем (шт) | Объем на 1 пользователя |
| :--- | :--- | :--- |
| Текстовые сообщения (история) | 2 678 | ~670 КБ (`2678 * 250 байт`) |
| Вложения (картинки, файлы) | 268 | ~268 МБ (`10% от сообщений * 1 МБ`) |
| Профиль | 1 | ~100 КБ |
| **Итого на пользователя** | | **~269 МБ** |

**Среднее количество действий пользователя в день**
1.  **Управление серверами**
    *   Лимит для бесплатного аккаунта - 100 серверов [^13]. Пользователи редко вступают в новые сервера или создают их. Примем частоту действия за 1 раз в неделю (0.14 в день).
2.  **Текстовые чаты (Отправка)**
    *   В день через платформу проходит **4 млрд сообщений** [^11]. В пересчете на активного пользователя: `4000 млн / 26.5 млн ≈ 150`.
    *   *Обоснование:* Цифра включает в себя сообщения от ботов и системные уведомления, которые инфраструктура обязана обработать и сохранить.
3.  **Голосовые каналы**
    *   Трафик составляет ~8 млрд минут в день (экстраполяция данных 2020 года [^6] на текущий MAU). На одного пользователя: `8000 млн / 26.5 млн ≈ 301 минута` (около 5 часов).
4.  **История сообщений (Чтение)**
    *   Пользователи открывают приложения мессенджеров в среднем 9 раз в день [^14].
    *   Согласно документации API Discord, при открытии канала клиент запрашивает пакет из **50 сообщений** по умолчанию [^15].
    *   Итого чтений (Loading): `9 сессий * 50 сообщений = 450 сообщений/день`.
5.  **Статусы и Fan-out**
    *   **Обоснование частоты (10 раз):** Складывается из сценария использования:
        1.  Вход/выход из приложения (2 события).
        2.  Игровая активность: согласно статистике [^16], средний геймер совершает **2.4 игровые сессии в день**. Каждая сессия генерирует статус "Начал играть" и "Закончил играть" (`2.4 * 2 ≈ 5` событий).
        3.  Переходы в режим "Не активен" (AFK) при перерывах (~3 события).
        4.  *Итог:* `2 + 5 + 3 = 10`.
    *   Учитывая архитектуру пассивных сессий [^9], сообщение доставляется не всем участникам сервера, а только активным. Примем эффективный fan-out = 20.
6.  **Вложения (Media)**
    *   Исходя из доли медиа-сообщений 10% [^12], пользователь загружает `150 * 0.1 = 15` файлов в день.

| Действие (MVP) | Количество в день | Обоснование |
| :--- | :--- | :--- |
| Управление серверами | 0.14 | 1 раз в неделю (лимит 100 серверов [^13]) |
| Отправка сообщений | 150 | Общая статистика (вкл. ботов) [^11] |
| Голосовая связь (минут) | 301 | Экстраполяция оф. данных [^6] |
| Чтение истории (сообщений) | 450 | 9 сессий [^14] * 50 msg limit [^15] |
| Смена статуса | 10 | Смена деятельности в течение дня |
| Загрузка вложений | 15 | 10% от отправленных сообщений [^12] |

### Технические метрики

**Обоснование пикового коэффициента (k = 1.5)**
Сервис является глобальным, поэтому нагрузка распределена по часовым поясам. Однако, согласно статистике трафика [^4], **~40%** аудитории сосредоточено в Северной и Южной Америке (США, Бразилия, Канада). Вечерний прайм-тайм этого макрорегиона создает основную волну нагрузки.
*   *Решение:* Используем коэффициент пиковой нагрузки **1.5**, так как остальные 60% трафика размазаны по Европе и Азии.

#### Размер хранилища
Discord хранит историю сообщений вечно. Расчет строится на накопленных данных (1.5 триллиона сообщений [^5]).

*   **Текстовые сообщения:** 1.5 трлн записей. Средний размер сжатого пейлоада - **250 байт** (по графику из инженерного блога [^10]).
*   **Медиа-вложения:** Согласно глобальной статистике мессенджеров (WhatsApp [^12]), доля сообщений с медиа составляет ~12%. Для игрового чата примем оценку **10%**.
    *   *Количество:* `1.5 трлн * 0.10 = 150 млрд файлов`.
    *   *Размер:* Лимит для бесплатных пользователей - 10 МБ [^8]. Средний размер файла (скриншоты, мемы) оценим в **1 МБ**.
*   **Метаданные серверов (Guilds):** 21 млн активных серверов [^2]. Храним роли, каналы, настройки. Средний размер объекта - 500 КБ.

| Тип данных | Количество (штук) | Расчет объема | Итоговый размер |
| :--- | :--- | :--- | :--- |
| **Текстовые сообщения** | 1.5 трлн | `1.5 * 10^11 * 250 байт` | **375 ТБ** |
| **Медиа-вложения** | 150 млрд | `150 * 10^9 * 1 МБ` | **150 ПБ** |
| **Профили пользователей** | 560 млн | `560 * 10^6 * 100 КБ` | **56 ТБ** |
| **Метаданные серверов** | 21 млн | `21 * 10^6 * 500 КБ` | **10.5 ТБ** |
| **Итого** | | | **~150.4 ПБ** |

#### Сетевой трафик
Расчет суточного и пикового потребления каналов связи.

*   **Голосовой трафик (Voice):**
    *   *Обоснование:* В 2020 году (100 млн MAU) трафик составлял 4 млрд минут/день [^6]. Экстраполируя на 200 млн MAU, получаем **8 млрд минут в день**.
    *   *Битрейт:* Стандартное качество - **64 kbps** (8 КБ/с) [^7].
*   **Медиа-трафик (CDN):**
    *   *Обоснование:* В сутки отправляется 4 млрд сообщений [^11]. 10% из них содержат файлы (400 млн файлов).
    *   *Объем:* `400 млн * 1 МБ = 400 ТБ` (загрузка). С учетом скачивания другими пользователями (Read factor ~3), суточный обмен ~1.2 ПБ.
*   **Текстовый Fan-out (WebSocket):**
    *   *Обоснование:* Сообщения доставляются через WebSocket. Благодаря архитектуре "пассивных сессий" [^9], сообщение не рассылается всем 1000 участникам сервера, а только тем, кто активен/смотрит экран. Примем эффективный **Fan-out = 20** (1 сообщение получают 20 человек).

| Тип трафика | Суммарный суточный (ГБ/сутки) | Средняя полоса (Гбит/с) | Пиковая полоса (Гбит/с) k=1.5 |
| :--- | :--- | :--- | :--- |
| **Голос (UDP)** | 3 840 000 | 355.5 | 533.3 |
| **Медиа (HTTP)** | 1 200 000 | 111.1 | 166.7 |
| **Fan-out (WS)** | 20 000 | 1.9 | 2.9 |
| **Итого** | **~5.06 ПБ/сут** | **~468.5 Гбит/с** | **~702.9 Гбит/с** |

#### RPS (Запросы в секунду)
Расчет нагрузки на бэкенд по типам запросов.
*   **Входящие сообщения (Write):** 4 млрд сообщений в сутки [^11].
    *   *Обоснование:* Эта цифра включает сообщения от ботов и системные уведомления.
*   **Чтение истории (Load History):**
    *   *Обоснование:* В среднем пользователь открывает приложение/меняет чат **9 раз в день** [^14].
    *   *Лимит:* По умолчанию API Discord запрашивает пакет из **50 сообщений** (`limit=50`) [^15].
    *   *Расчет:* `26.5 млн DAU * 9 сессий = 238.5 млн запросов`.
*   **Доставка (Push):** Работает через WebSocket push.
    *   *Расчет:* `Write RPS * Fan-out (20)`.

| Тип запроса | Обоснование расчета (сутки / 86400) | Средний RPS | Пиковый RPS (x1.5) |
| :--- | :--- | :--- | :--- |
| **Отправка сообщения** (Write) | `4 млрд / 86400` | 46 296 | 69 444 |
| **Доставка / Fan-out** (Push) | `Write RPS * 20` | 925 920 | 1 388 880 |
| **Загрузка истории** (Read API) | `(26.5 млн * 9) / 86400` | 2 760 | 4 140 |
| **Загрузка файлов** (Upload) | `10% от Write RPS` | 4 630 | 6 945 |
| **Обновление статуса** (Presence) | `(26.5 млн * 10) / 86400` | 3 067 | 4 600 |
| **Управление серверами** | `(26.5 млн * 0.14) / 86400` | 43 | 65 |
