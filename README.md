## 1. Тема и целевая аудитория
**Discord** - платформа для текстового и голосового общения в рамках пользовательских сообществ (серверов).

### Целевая аудитория
*   **MAU (Monthly Active Users):** 200 млн [^1].
*   **DAU (Daily Active Users):** 26.5 млн [^3].
*   **Количество зарегистрированных аккаунтов:** 560 млн [^2].
*   **География:** Глобальный рынок. Распределение трафика по данным на начало 2026 года [^4]:

| Страна | Доля трафика (%) | Оценка MAU (млн чел) |
| :--- | :--- | :--- |
| США | 29.1% | 58.2 |
| Бразилия | 7.25% | 14.5 |
| Индия | 4.9% | 9.8 |
| Германия | 3.2% | 6.4 |
| Канада | 2.7% | 5.4 |

### Функционал MVP
Основной функционал заключается в создании иерархических сообществ и обеспечении каналов связи внутри них.

1.  **Управление серверами:** создание серверов и текстовых/голосовых каналов.
2.  **Текстовые чаты:** обмен сообщениями в реальном времени внутри каналов.
3.  **Голосовые каналы:** аудио-сессии для группового общения.
4.  **История сообщений:** хранение и доступ к переписке на стороне сервера.
5.  **Статусы:** статусы пользователей («в сети», «играет в...»).
6.  **Вложения (Media):** возможность загрузки изображений и файлов в текстовые каналы.

### Ключевые продуктовые решения
*   Голосовые каналы работают по модели «постоянной комнаты».
*   История сообщений хранится на серверах бессрочно и доступна с любого устройства в любой момент.
*   Синхронизация состояния серверов, сообщений и статусов между всеми активными сессиями пользователя.

## 2. Расчет нагрузки

### Продуктовые метрики
Базовые показатели аудитории, на которые мы опираемся в расчетах:
*   **MAU (Monthly Active Users):** 200 млн [^1]
*   **DAU (Daily Active Users):** 26.5 млн [^3]
*   **Зарегистрированных аккаунтов:** 560 млн [^2]
*   **Активных серверов (Guilds):** 21 млн [^2]

**Средний размер хранилища пользователя**
Официально Discord хранит более 1.5 триллиона сообщений [^5]. Чтобы найти среднее количество сообщений на пользователя, делим общее число на количество зарегистрированных аккаунтов: `1.5 трлн / 560 млн ≈ 2678 сообщений`.
*   *Текст:* Одно текстовое сообщение в среднем весит около 300 байт.
*   *Вложения:* Лимит бесплатной загрузки составляет 10 МБ [^8], но средний размер картинки/документа примем за 1 МБ. Доля сообщений для чатов примем ~5% (около 134 медиафайлов на пользователя).
*   *Профиль:* Состоит из аватара (~98 КБ) и текстовых настроек (~2 КБ).

| Тип данных | Объем (шт) | Объем на 1 пользователя |
| :--- | :--- | :--- |
| Текстовые сообщения (история) | 2 678 | ~803 КБ (`2678 * 300 байт`) |
| Вложения (картинки, файлы) | 134 | ~134 МБ (`134 * 1 МБ`) |
| Профиль (аватар + настройки) | 1 | ~100 КБ |
| **Итого на пользователя** | | **~135 МБ** |

**Среднее количество действий пользователя в день**
Согласно статистике, ежедневно отправляется 850 млн сообщений [^2]. Активность одного пользователя считаем относительно DAU (26.5 млн): `850 млн / 26.5 млн ≈ 32.1 сообщения в день`. 
Голосовой трафик в 2020 году составлял 4 млрд минут в день [^6] при 100 млн MAU. Пропорционально масштабируя на текущие 200 млн MAU, получаем ~8 млрд минут в день. На одного активного пользователя: `8 млрд / 26.5 млн ≈ 301.8 минут` (около 5 часов).

| Тип действия | Среднее количество в день на активного пользователя (DAU) |
| :--- | :--- |
| Отправка сообщения | 32.1 |
| Отправка вложения (5% от сообщений) | 1.6 |
| Просмотр истории (чтение) | 20.0 |
| Подключение к голосовому каналу | 2.0 |
| Пребывание в голосовом чате | 301.8 минут |

### Технические метрики

**Размер хранения (по типам данных)**
Расчет суммарного объема накопленных данных:
*   *Медиа-вложения:* `1.5 трлн сообщений * 5% = 75 млрд файлов`. При среднем размере 1 МБ это 75 ПБ.
*   *Сервера:* 21 млн активных серверов [^2]. Средний размер метаданных сервера (иконка, список ролей, список каналов) оцениваем в 500 КБ.

| Тип данных | Количество записей | Общий размер |
| :--- | :--- | :--- |
| Текстовые сообщения | 1.5 трлн[^5] | 450 ТБ |
| Медиа-вложения | 75 млрд | 75 ПБ |
| Пользовательские профили | 560 млн | 56 ГБ (`560 млн * 100 КБ`) |
| Метаданные серверов | 21 млн | 10.5 ГБ (`21 млн * 500 КБ`) |
| **Итого** | | **~75.45 ПБ** |

**Сетевой трафик**
Для расчета берем стандартный битрейт аудио 64 kbps (8 КБ/с) [^7]. Трафик считаем исходя из 8 млрд минут в день: `8 млрд * 60 сек * 8 КБ/с = 3.84 ПБ/сутки`.
Текстовые сообщения рассылаются через WebSocket. Согласно блогу Discord [^9], используется архитектура Fan-out (вещание всем участникам онлайн). Примем средний Fan-out за 50 (сообщение доставляется 50 участникам канала).
Пиковое потребление рассчитываем с коэффициентом запаса `k = 2`.

| Тип трафика | Суммарный суточный | Средняя пропускная способность | Пиковое потребление |
| :--- | :--- | :--- | :--- |
| Голосовая связь (UDP, 64 kbps) | 3 840 000 ГБ/сут | 355.5 Гбит/с | 711.0 Гбит/с |
| Медиа и файлы (HTTP, 1 МБ/файл) | 42 500 ГБ/сут | 3.9 Гбит/с | 7.8 Гбит/с |
| Fan-out сообщений (WS, 300 байт) | 12 750 ГБ/сут | 1.2 Гбит/с | 2.4 Гбит/с |
| **Итого** | **~3.89 ПБ/сут** | **~360.6 Гбит/с** | **~721.2 Гбит/с** |

**RPS (запросов в секунду)**
Средний RPS рассчитывается по формуле: `Суммарное количество запросов в сутки / 86400 секунд`. Пиковый RPS равен `Средний RPS * 2`. 
Ежедневно в Discord блокируется огромное количество спама (35 млн аккаунтов за полгода [^10]). Это означает, что реальная нагрузка на запись (Write RPS) может иметь резкие всплески от бот-сетей, поэтому берем запас x2 .

| Тип запроса | Логика расчета (в сутки / 86400) | Средний RPS | Пиковый RPS |
| :--- | :--- | :--- | :--- |
| Отправка сообщения (Write) | `850 млн / 86400` | 9 838 | 19 676 |
| Доставка сообщений (Push/Fan-out) | `(850 млн * 50) / 86400` | 491 898 | 983 796 |
| Загрузка вложений (Media Upload) | `42.5 млн / 86400` | 492 | 984 |
| Чтение истории (Read) | `(26.5 млн DAU * 20) / 86400` | 6 134 | 12 268 |
| Смена статусов (Presence) | `(26.5 млн DAU * 10) / 86400` | 3 067 | 6 134 |
